# 高频追问方向

> 来源：王子亭博客, CSDN, JavaGuide, EightShapes

## 面试官视角的追问策略

| 策略 | 说明 |
|------|------|
| 围绕同一话题纵深 | 准备难度递增的多个问题，避免零散无关的小问题 |
| 基于候选人优势提问 | 让对方在擅长话题上发挥正常水平 |
| 验证项目真实性 | 聊简历上的项目，请对方介绍业务、职责、遇到的问题 |
| 灵活应变 | 候选人多次提示后仍未答对，不必继续纠缠 |

## 六大追问模式

1. **展开模式**："能多讲讲这部分吗？"
2. **视角转换模式**：让候选人从不同角度审视同一问题
3. **举例模式**："能再给一个例子吗？"
4. **挑战假设模式**：表面化解答的深层挑战
5. **关联模式**：将看似无关的话题关联起来
6. **对比模式**："这和你之前的做法有什么不同？"

## 项目面试常见追问方向

### 技术选型理由
每当候选人提到使用了某个技术/框架/工具：
- "为什么选了 X 而不是 Y？"
- "你比较了哪些方案？"
- "现在回头看，这个选择合适吗？"

### 困难与解决
最能考察真实经验的追问方向：
- "做这个项目遇到的最大技术难题是什么？"
- "你是怎么解决的？中间走过弯路吗？"
- "解决方案有没有什么妥协或不完美的地方？"

### 如果重来
考察反思能力和架构视野：
- "如果重新做这个项目，你会改什么？"
- "架构上有没有你觉得不够好的地方？"

### 项目真实性验证
- 对比简历与口述的一致性
- 追问具体细节（时间、团队规模、个人负责模块）
- 从不同角度问同一个问题，检查是否矛盾

### 规模与扩展
- "这个系统目前的用户量/数据量大概是多少？"
- "如果规模增长 10 倍，哪里会先出问题？"
