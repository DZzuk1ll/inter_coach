# 后端岗位考察侧重

> 来源：Yardstick, EliteBrains, TestGorilla

## 五阶段面试评估

| 阶段 | 考察重点 | 示例问题 |
|------|---------|---------|
| 筛选面试 | 基础技术能力、技术栈匹配 | "你最熟悉哪些技术和编程语言？" |
| 技术评估 | 编码实战能力 | 实现 REST API 的 CRUD、错误处理、数据验证 |
| 技术深度面试 | 技术决策和实现深度 | "解释你的技术方案选择及其权衡" |
| 系统设计面试 | 架构能力 | "为这个系统设计后端架构" |
| 团队文化面试 | 协作与沟通 | "描述一次技术分歧是如何解决的" |

## 核心评估维度

1. **核心技术能力**：数据结构与算法、数据库设计、API 设计（REST/GraphQL）、安全原则、测试思维
2. **问题解决与系统思维**：将复杂问题拆解为可管理任务的能力、理解系统各部分如何交互
3. **技术适应性**：可迁移的编程概念（OOP、错误处理）、独立学习能力
4. **行为与认知能力**：好奇心和成长导向型心态、模糊场景下的问题解决、沟通协作

## 五大工作目标导向评估

1. 构建性能可靠的系统（生产问题最小化）
2. 创建易集成的 API（完善的文档）
3. 改进开发流程（测试覆盖、CI/CD）
4. 减少技术债务（重构与架构优化）
5. 指导初级开发者（知识共享）

## 评分标准（0-4 分制）

| 分数 | 含义 |
|------|------|
| 0 | 信息不足以评估 |
| 1 | 显著不足/有重大缺陷 |
| 2 | 基础/有改进空间 |
| 3 | 强能力/符合要求 |
| 4 | 卓越/超出预期 |

## 高频追问方向
- "这个接口的 QPS 大概多少？有做过压测吗？"
- "数据库事务在哪些场景下使用了？隔离级别是什么？"
- "如果某个下游服务挂了，你的系统怎么处理？"
- "日志是怎么记录的？出了问题怎么排查？"
